## About Bario Entertainment System (BAES)

**BAES** (Bario Entertainment System) is a Web3 gaming platform built on **Base chain** that revolutionizes how game developers distribute and monetize their games.

### Why BAES is Different from Traditional Game Stores

| Feature | Traditional Stores (Steam, Epic, itch.io) | BAES |
|---------|-------------------------------------------|------|
| Revenue Share | 70% to developer (30% platform cut) | **90% to developer** |
| Payment Speed | 30-90 day hold periods | **Instant payments** |
| Ownership | License that can be revoked | **NFT = true ownership forever** |
| Storage | Platform servers (can disappear) | **IPFS pinned (permanent, decentralized)** |
| Resale | Not allowed / No dev benefit | **Secondary market + 10% royalty forever** |
| Supply Control | Unlimited copies | **You set the max supply (scarcity)** |

### How Ownership Works (Technical)

```
Traditional: Buy game ‚Üí Get license ‚Üí Platform can revoke ‚Üí Files on their servers

BAES: Buy game ‚Üí Receive NFT ‚Üí You OWN it forever ‚Üí Files on IPFS (permanent)
      ‚Üì
      Can resell on secondary market
      ‚Üì
      Original creator gets 10% of every resale
```

**IPFS (InterPlanetary File System):**
- Decentralized storage - no single point of failure
- Content-addressed - files identified by their hash, not location
- Permanent - as long as someone pins it, it exists
- Your game files live forever, even if BAES disappears

**NFT (Non-Fungible Token):**
- Proof of ownership on Base blockchain
- Transferable - can be sold, gifted, traded
- Verifiable - anyone can check who owns what
- Smart contract enforces royalties automatically

**Gasless Transactions:**
- BAES uses account abstraction
- Platform sponsors all gas fees
- Buyers experience Web2-like simplicity
- No wallet pop-ups asking for gas approval

### Creating a Great Cartridge (Cover Art)

The cartridge is your game's "box art" - it's the first thing players see!

**Tips:**
- Use bold, readable text for the title
- Choose colors that stand out in a grid of games
- Show a key character, scene, or logo
- Keep it simple - it will be displayed small

**What to avoid:**
- Tiny text that's unreadable at small sizes
- Too much visual clutter
- Generic stock art
- Screenshots as cover art (use them in the gallery instead)

### Screenshot Best Practices

Screenshots sell your game! Here's how to make them great:

**Do:**
- Capture actual gameplay (not menus)
- Show variety - different levels, mechanics, moments
- Use native resolution for crisp images
- Capture exciting action or beautiful scenes
- Include UI if it's part of the experience

**Don't:**
- Use blurry or low-resolution images
- Show bugs or placeholder art
- Include personal information or debug text
- Upload the same screenshot multiple times

**Recommended order:**
1. Most exciting/impressive shot first
2. Core gameplay demonstration
3. Different environments/levels
4. Unique features or mechanics
5. Character/customization (if applicable)

### Security Pattern Errors (Malicious Code Detection)

The system scans all text files for dangerous patterns:

**Blocked Patterns (Errors):**
- Data URI script injection: `<script src="data:text/javascript`
- JavaScript eval in URLs: `javascript:eval`
- VBScript: `vbscript:`
- Java applets: `<applet>`
- ActiveX shells: `new ActiveXObject("Shell")`
- Shell execution: `shell.exec`, `shell.run`, `shell.spawn`
- Windows Script Host: `WScript.Shell`
- System commands: `cmd.exe /c`, `powershell -command`
- Meta refresh redirects: `<meta http-equiv="refresh" url=`
- Domain manipulation: `document.domain =`

**Warning Patterns (Allowed but flagged):**
- `eval()` usage (common in game engines)
- `ActiveXObject` (common in polyfills)
- `document.location` redirects
- `top.location` frame breakouts

## Frequently Asked Questions

### Account & Wallet

**Q: Do I need a crypto wallet to use BAES?**
A: No! You can sign in with Twitter or Email, and BAES creates an embedded wallet for you automatically. If you already have a wallet (MetaMask, Rainbow, etc.), you can connect that instead.

**Q: Do I need cryptocurrency to publish a game?**
A: No! Publishing is free and BAES sponsors all gas fees. You just need to sign in.

**Q: Can I sign in with Twitter?**
A: Yes! Click sign in, choose Twitter, authorize the connection, and you're done. An embedded wallet is created for you automatically.

**Q: What's the difference between Twitter login and wallet login?**
A: Both work the same way once you're in. Twitter/Email login creates an embedded wallet for you (managed by Privy). Wallet login uses your existing EOA (MetaMask, etc.). Your earnings go to whichever wallet is linked to your account.

**Q: Can I use multiple wallets?**
A: Each developer profile is linked to one wallet/account. Use the same login method consistently.

**Q: What if I lose access to my account?**
A: If you used Twitter/Email login, you can recover via that same method. If you used an EOA wallet, keep your seed phrase safe - if lost, you'll need to create a new developer account.

### Publishing

**Q: How long does review take?**
A: Typically 24-72 hours. Complex games may take longer.

**Q: Can I publish multiple games?**
A: Yes! There's no limit to how many games you can publish.

**Q: Can I unpublish a game?**
A: Yes, you can revert a published game to draft status from your dashboard.

**Q: What file formats are supported?**
A: HTML5/WebGL games only. Your game must have an index.html file that runs in a browser.

### Revenue & Payments

**Q: How do I get paid?**
A: Payments go directly to your connected wallet in ETH. No waiting periods.

**Q: What's the revenue split?**
A: You keep 90% of all sales. The protocol takes 10% to fund discovery, distribution, liquidity, and platform growth.

**Q: Do I earn from resales?**
A: Yes! You earn perpetual royalties whenever your game NFT is resold.

**Q: What is $BAES and how do I earn it?**
A: $BAES is the platform's growth equity token. Beyond your 90% sales revenue, you can earn $BAES by building products that grow the platform - games with meaningful usage, healthy MAU, and good retention. These rewards are bought from the open market, not minted.

**Q: Is $BAES inflationary?**
A: No. $BAES has a fixed supply with 100% in circulation. No mint, no unlock, no inflation. All creator rewards are funded by open market purchases.

### NFTs & Ownership

**Q: What does it mean that games are NFTs?**
A: When someone buys your game, they receive an NFT (Non-Fungible Token) that proves ownership. This isn't a revocable license - it's true ownership that lives on the blockchain forever.

**Q: Where are the game files stored?**
A: Game files are pinned to IPFS (InterPlanetary File System), a decentralized storage network. This means your game exists permanently, independent of any single server or company.

**Q: Can buyers resell games they purchased?**
A: Yes! Unlike traditional platforms, players can resell their game NFTs on secondary markets. This creates liquidity and collectibility for your games.

**Q: How do secondary sale royalties work?**
A: Every time your game NFT changes hands (is resold), you automatically receive 10% of the sale price. This is enforced by smart contract - it happens automatically, forever, without any action needed from you.

**Q: Who pays gas fees?**
A: Nobody! BAES sponsors all gas fees using account abstraction. Buyers experience a seamless Web2-like purchase flow, and you never worry about gas costs.

**Q: Can I change the max supply after publishing?**
A: No. Max supply is permanent once set. You can never mint more copies than your original limit. Choose carefully - scarcity drives value.

### Technical

**Q: What's the maximum game size?**
A: 500MB compressed. Optimize your assets for the best player experience.

**Q: Can I use external APIs?**
A: Yes, but be aware that some players may have connectivity issues.

**Q: Do mobile browsers work?**
A: Yes! You can specify mobile, desktop, or both platform support.

## Error Recovery Guide

### "Wallet address verification failed"
**What happened:** Session and wallet are out of sync.
**Solution:**
1. Click disconnect in wallet dropdown
2. Refresh page
3. Reconnect wallet
4. Re-sign authentication message

### "Session expired"
**What happened:** You were inactive too long or cookies were cleared.
**Solution:**
1. Refresh page
2. If still disconnected, reconnect wallet
3. Re-authenticate

### "ZIP security validation failed"
**What happened:** Your game contains files that triggered security checks.
**Solution:**
1. Read the specific error message carefully
2. If it mentions a file, check that file
3. Remove blocked file types (.exe, .dll, etc.)
4. Re-create the ZIP and upload again

### "This URL is already taken"
**What happened:** Another developer registered that slug.
**Solution:**
1. Try adding numbers: `myname1`, `myname2`
2. Try variations: `mygames`, `mystudio`
3. Use initials or abbreviations
4. Get creative with your brand!

### Upload stuck or failed
**What happened:** Network issues or server timeout.
**Solution:**
1. Check internet connection
2. For large files (100MB+), wait longer (can take 10+ minutes)
3. If failed, try a smaller file or compress assets
4. Try a different browser if issues persist
5. Check browser console for specific errors

### Game stuck in "unlisted" status
**What happened:** Game is in the review queue.
**Solution:**
- This is normal - review takes 24-72 hours
- Contact support via Discord if longer than 72 hours
- You cannot edit while in review

### "Missing index.html"
**What happened:** Your ZIP doesn't have index.html at the root level.
**Solution:**
1. Unzip your file
2. Make sure index.html is at the TOP level (not in a subfolder)
3. Re-create the ZIP by selecting all files, not the parent folder
